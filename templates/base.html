Tools & Technologies Used

  • HTML5 – For the basic structure of all pages.
  • SCSS (SASS) – To write clean, modular, and maintainable styles with nesting, mixins and variables.
  • JavaScript – For interactivity, and dynamic behaviors.
  • AOS (Animate On Scroll) – To enhance the user experience with subtle entry animations.
  • Go (Golang) Templates – Used for backend templating logic and serving dynamic HTML pages.
  • Git & GitHub – For version control.
  • Render – To deploy and host the backend server for the live portfolio.

Purpose of This Project

This portfolio was built to present my skills as a full-stack web designer and developer. It serves as both a personal showcase and a real-world demonstration of my ability to:

  • Build responsive web layouts.
  • Present my resume and contact information.
  • Demonstrate both front-end and back-end abilities.

Key Features

  • Mobile-first design that works smoothly on all devices.
  • Scroll-triggered animations for better visual engagement.
  • Contact form to allow users to reach out.
  • Reusable SCSS components for maintainable and scalable styles.
  • Code structure that separates concerns and follows clean, semantic practices.

What I Learned

  • How to debug tricky layout issues like horizontal overflow and responsiveness.
  • The importance of component structure and using SCSS variables and mixins efficiently.
  • Integrating JavaScript libraries and managing third-party dependencies.
  • Using Go’s html/template package for rendering content safely and efficiently.
  • How to deploy a full-stack app on Render.
